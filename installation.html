Installation

Install BIDS Manager

Two paths. A one-double-click bootstrap installer for each OS that pulls a portable Python 3.10, builds an isolated environment, and registers a native launcher. Or pip install bids-manager in any Python 3.10 to 3.14 you already have.

▸
No system Python required.

The bootstrap installer downloads its own portable Python so nothing on your machine is changed. Uninstall is a single double-click and removes everything cleanly.

Recommended

Bootstrap installer

One ZIP for all three OSes. Inside is one folder per platform with a single script to run.

Download Installers.zip

Unzip it, open the folder for your operating system, and follow the steps below.

The macOS installer is built for Apple Silicon (arm64). On Intel Macs, use the pip install path instead.

  1. Double-click install_BIDS_Manager.command inside the MacOS/ folder.
  2. macOS shows an unidentified developer warning. Click Done. The OS has now seen the file and lets you whitelist it.
  3. Open System Settings → Privacy & Security, scroll to the Security section, and click Open Anyway next to the blocked script.
  4. Re-double-click the script. A terminal opens and the install runs (3 to 5 minutes on a typical connection).
  5. When it finishes, launch BIDS Manager from Applications, Launchpad, or Spotlight. The app icon is registered in ~/Applications/BIDS-Manager.app.
▸
Prefer the terminal?

Open Terminal in the MacOS/ folder and run ./install_BIDS_Manager.command. Same result, no Gatekeeper dialog.

The Linux installer is built for x86_64. It registers BIDS Manager in your application menu with the app icon, so it can be launched like any other app afterwards.

  1. Open a terminal in the Linux/ folder (right-click in the file manager → Open terminal here, or cd manually).
  2. Make the script executable. 
    chmod +x install_BIDS_Manager.sh
  3. Run it. 
    ./install_BIDS_Manager.sh
  4. When it finishes, launch BIDS Manager from your application menu or run bidsmgr in any terminal.
◆
XFCE / Thunar users.

By default Thunar refuses to execute shell scripts on double-click. Either run the script from a terminal (as above), or enable script execution once with: 

xfconf-query --channel thunar \
  --property /misc-exec-shell-scripts-by-default \
  --create --type bool --set true

The Windows installer is built for x86_64 (Windows 10 and Windows 11). It places a Desktop shortcut and a Start Menu entry with the BIDS Manager icon.

  1. Inside the unzipped Windows/ folder, double-click install_BIDS_Manager.bat.
  2. SmartScreen may show a blue Windows protected your PC banner. Click More info → Run anyway.
  3. A console opens and the install runs. When it finishes, a BIDS-Manager shortcut appears on your Desktop and in the Start Menu.
  4. Double-click the Desktop shortcut to launch.
▸
First launch is slower.

The first time the app starts, it pre-compiles the embedded Python and caches the BIDS schema. Subsequent launches are fast.

What does the installer do?
  1. Creates a BIDS-Manager/ folder in your home directory.
  2. Downloads a portable Python 3.10 build (Apple Silicon arm64, Linux x86_64, or Windows embeddable depending on your OS).
  3. Extracts it under ~/BIDS-Manager/ so nothing touches your system Python.
  4. Creates a virtual environment with that Python.
  5. Runs pip install bids-manager inside the environment (pulls in PyQt6, mne-bids, dcm2niix, pydicom, bidsschematools, and the rest).
  6. Registers a native launcher: a Desktop and Start Menu shortcut on Windows, a .app bundle under ~/Applications/ on macOS, an application-menu entry with the app icon on Linux.
  7. Writes an uninstaller alongside the launcher.

BIDS Manager is pinned to Python 3.10 inside the bundle so every installation behaves identically. The Python you have on your system is left untouched.

Alternative

pip install (existing Python)

If you already manage a Python environment (conda, venv, uv, pyenv, whatever), install BIDS Manager directly. Supported on Python 3.10 through 3.14. 

# in a fresh virtual environment
pip install bids-manager
bidsmgr

PyPI: pypi.org/project/bids-manager. The package name is bids-manager; the Python import name is bidsmgr (same pattern as pip install scikit-learn / import sklearn).

Command-line use

Activate the bundled environment to get the CLI on your PATH. The installer set it up at ~/BIDS-Manager/env/. 

source ~/BIDS-Manager/env/bin/activate

source ~/BIDS-Manager/env/bin/activate

%USERPROFILE%\BIDS-Manager\env\Scripts\activate

Once activated, six commands are available:

  • bidsmgr. Launch the GUI (Converter and Editor in one window).
  • bidsmgr-scan. Walk a raw folder and produce a 51-column inventory TSV.
  • bidsmgr-rebuild. Re-derive BIDS names from the TSV after manual edits.
  • bidsmgr-convert. Run the conversion to BIDS for the rows in the TSV.
  • bidsmgr-metadata. Run the post-conversion metadata fix-ups (field-maps, IntendedFor, scans.tsv).
  • bidsmgr-validate. Two-layer validation: BIDS Manager's schema audit plus bidsschematools.

The GUI uses the same engine; switching between GUI and CLI is frictionless.

Troubleshooting

If something goes wrong

macOS: "unidentified developer" / app cannot be opened

Apple's Gatekeeper blocks unsigned scripts on first run. After you see the dialog, open System Settings → Privacy & Security, scroll to the Security section, and click Open Anyway. Re-run the installer. This is a one-time approval.

macOS: installer runs on Intel Mac

The bundled macOS Python is arm64-only. On Intel Macs, install via pip in a Python 3.10+ environment instead. See pip install (existing Python).

Linux: "could not load the Qt platform plugin xcb"

PyQt6 needs a system Qt platform library. On Debian / Ubuntu: 

sudo apt install libxcb-cursor0 libxcb-xinerama0 libfontconfig1

On Fedora: 

sudo dnf install xcb-util-cursor fontconfig
Linux: Thunar refuses to run the script on double-click

By design. Run from a terminal, or enable script execution once with the xfconf-query command from the Linux tab above.

Windows: SmartScreen blocks the installer

Click More info → Run anyway. The installer is not code-signed yet, so SmartScreen flags any unknown publisher. The download still has to come from the official GitHub release link to be safe.

Windows: update fails with "file in use"

Close BIDS Manager fully before updating. From v1.0.2 the in-GUI updater spawns a detached helper that handles this automatically, but a manual pip install --upgrade still needs the app closed.

Any OS: install log

Every installer writes ~/BIDS-Manager/install.log. Paste it into a GitHub issue if you need help.
