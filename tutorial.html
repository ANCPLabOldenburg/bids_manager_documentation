Technique B: simulated widgets.

Each scene re-creates the active panel in HTML / CSS using the same palette tokens as the GUI itself. Click the controls; they will behave like the real thing on a short script. The CLI counterpart for each scene is shown alongside.

The actual interactive scenes (scan, inspect, convert, edit, validate) are authored in the next pass.

Planned scenes

  1. Set raw + BIDS folders with the path bars.
  2. Run a scan: watch the spinner + status chips update as the inventory builds.
  3. Inspect the table: spotlight the columns, tooltip explanations on hover.
  4. Override a row: edit a cell, see the BIDS preview update.
  5. Bulk-edit: multi-select rows, apply one value across the selection.
  6. Run conversion: per-subject progress animation, atomic rename, status chips.
  7. Open the result in Editor: BIDS tree, sidecar form pane, schema-driven required / recommended / optional colour legend.
  8. Inspect a NIfTI: tri-view, crosshair scrub, 4-D graph for BOLD.
  9. Validate the dataset: severity chips, file vs folder vs dataset scope, strict BIDS toggle.

Each scene will pair with its CLI equivalent (bidsmgr-scan, bidsmgr-convert, bidsmgr-validate, etc.) so power users can read both side by side.
