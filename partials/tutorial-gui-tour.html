GUI tour

The window you'll be working in.

BIDS Manager opens on a Welcome screen where you create or open a dataset project, then gives you two working views: the Converter (raw → BIDS) and the Editor (read, fix, validate). Pick a tab below, then hover or click a numbered marker to learn what each part of the window does. To watch these features in action, head to the GUI tour.

Click any number on the screenshot to see what that section does. Click again (or press Esc) to dismiss.

View switcher

The three top-level tabs that run the whole app: Home (this Welcome / project screen), Converter (raw → BIDS), and Editor (read, fix, validate). The Converter and Editor stay disabled until a project is open, which is why the workflow always starts here.
Create a new dataset

Type a name and pick a location, and BIDS Manager scaffolds a fresh BIDS dataset: dataset_description.json, a README, and a .bidsignore, plus the project bundle that records your work. The folder name becomes the dataset slug used throughout. This is the usual starting point.
Open an existing dataset

Continue a BIDS Manager project exactly where you left off, or adopt a dataset that was created elsewhere. An adopted dataset is opened read-only and is never overwritten, so you can browse, view, and validate it with no risk of changing the files.
Recent projects

Every project you have created or opened, each with its full path. Double-click one to reopen it; right-click to remove it from the list or delete it from disk. This list, plus the saved project bundle, is what makes curation resumable across sessions.
Getting started

Shortcuts to the documentation, this tutorial, and the source code, plus four downloadable sample datasets (MRI, EEG, MEG, and an advanced MRI set) so you can run the full scan → convert → validate loop on real data without hunting for files first.
Settings + theme

The gear opens Settings (Display, System, Scan, Scan rules, and Convert + post-convert). The sun / moon button beside it toggles the light / dark theme live. Both sit at the far right of the header on every screen.

View switcher

The Home / Converter / Editor tabs. You're in the Converter, where the raw folder is turned into a curated BIDS dataset.
Scan

The Scan… button walks the raw input folder and builds the inventory; the Scan table dropdown beside it names the current scan. Re-scanning saves a new versioned snapshot inside the project instead of overwriting the previous one, so earlier scans stay recoverable.
Project switcher + Undo / Redo

The dropdown shows the active dataset and moves between open projects (or back to Home). Undo and Redo in the toolbar below step through your curation edits; because every change is recorded in the project bundle, nothing is ever lost.
Status chips + Re-validate

Four chips summarise the inventory, valid, warnings, error, and skipped, and update live as a scan runs. Three are clickable to list the rows behind them. Re-validate re-checks every row's entities after you edit them.
Run conversion

Writes the curated rows to the project's BIDS root, running the right backend per modality. It stays disabled until a scan has finished, and flips to Cancel while a conversion is in progress.
Settings + theme

The gear opens Settings (Display, System, Scan, Scan rules, Convert + post-convert), where you tune workers, the classifier, scan rules, and the post-conversion chain. The sun / moon button toggles the theme live.
PathBars

Raw input is the folder you scan. BIDS output is locked to the open project, so it cannot be pointed at the wrong place. The glyph at the left of each bar flips from a circle to a check when the path is valid.
Raw & output trees

The top tree browses the raw input folder exactly as it sits on disk (a read-only navigator). The bottom tree previews the future BIDS layout the conversion will write, including dataset_description.json, participants.tsv, and each subject folder.
Filter / structure

Every series grouped by schema entity (subject / session / datatype) with tri-state checkboxes. Filter by entity to focus the Inspection table on a subset, or untick a whole group to exclude those rows from the next conversion.
Inspection table

The centre of gravity: one editable row per series, with the predicted basename, datatype, task, run, dataset, and confidence. Rows are colour-coded by status (keep / skip / non-image / warning). The classifier's guesses are only defaults, every cell can be edited.
Curation actions

The buttons beneath the table: Dataset metadata opens the dataset-wide editor (including the EEG / MEG enrichment: reference, ground, filters, device, event labels); Manage columns shows, hides, and reorders columns; Bulk edit sets an entity on many selected rows at once.
Properties panel

A schema-aware editor for the selected row. It shows only the entities valid for that row's datatype + suffix, marks required fields with a red asterisk, and updates the predicted path as you type. EEG / MEG rows also expose minimal metadata and per-row device overrides.
BIDS preview & log dock

The bottom dock with tabs: Log (engine output during scan and conversion), BIDS preview (the exact filesystem layout that will be written), and Statistics.

View switcher

You're in the Editor: read the converted BIDS dataset, edit sidecars and TSV tables, inspect volumes and recordings, and run validation.
Open BIDS root + Undo / Redo

Open BIDS root picks the dataset to edit (the active project opens here automatically). Undo and Redo step through sidecar and table edits.
Validation toolbar

Validate file / folder / dataset run audits at three scopes. The Strict BIDS toggle adds the official bidsschematools structural pass on top of the schema-driven checks.
Project switcher + severity chips

The same project dropdown as the Converter. Once you run a validation, severity chips appear here too, summarising the dataset's current issue counts, and are clickable to open the issue list.
Settings + theme

The gear opens the same Settings dialog as the Converter, and the sun / moon button toggles the light / dark theme live.
BIDS dataset tree

The converted layout with a status dot per file: green for clean, amber for warnings, red for errors. Click a file to open it in the centre pane; the dots refresh after you re-validate.
File viewer (extension-routed)

The centre pane routes by file type: JSON sidecars open a schema-aware form (with a Tree view, Add / Delete field, and Revert / Save); TSVs open an editable table; NIfTI opens the tri-view + 4-D graph; and MEG / EEG / iEEG recordings open the interactive signal viewer with an in-app PSD and an events overlay.
Validation pane

The severity-coloured issue list. Click an item to jump to the offending file; fix it in the viewer and re-run validation to refresh the list and the tree's status dots.
